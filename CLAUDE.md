# Описание проекта: Family Web Planner MVP 1.0

## 🎯 Главная директива
Ты — Senior Full-Stack разработчик. Твоя задача — пошагово создать закрытое веб-приложение для семьи согласно техническому заданию, описанному в файле `tz.md`. 
**КРИТИЧЕСКИ ВАЖНО:** Мы делаем ТОЛЬКО функционал MVP (Версия 1.0). Никакой интеграции с Telegram, никаких сложных ролей, никаких дашбордов с аналитикой, пока это явно не попросит пользователь. Делай максимально простой, читаемый и рабочий код. Не пиши код "на будущее", если он усложняет текущую архитектуру.

## 🛠 Стек технологий
- **Фреймворк:** Next.js (App Router).
- **Язык:** TypeScript (строгая типизация обязательна, используй интерфейсы для всех сущностей БД).
- **Стилизация:** Tailwind CSS.
- **UI Компоненты:** shadcn/ui (используй готовые компоненты для кнопок, инпутов, чекбоксов, карточек).
- **Backend & DB & Auth:** Supabase (используй @supabase/supabase-js).
- **Иконки:** Lucide React.

## 🛑 Строгие правила разработки
1. **Mobile-First:** Весь UI должен идеально смотреться на экране смартфона. Делай крупные элементы тач-зоны (особенно чекбоксы и кнопки).
2. **KISS (Keep It Simple, Stupid):** Не создавай сложную глобальную систему стейт-менеджмента (Redux/Zustand), если можно обойтись локальным стейтом React и хуками Supabase.
3. **Realtime:** Использование Supabase Realtime для синхронизации списков между устройствами — приоритет.
4. **Контекст:** Прежде чем писать код, всегда читай `tz.md` для понимания бизнес-логики текущего модуля.

## 📋 Пошаговый план разработки (Roadmap)

Жди команды пользователя для перехода к каждому следующему этапу. Не пытайся сделать всё за один раз.

### Этап 1: Инициализация и Базовый UI
- Настроить Next.js проект с Tailwind и TypeScript.
- Инициализировать shadcn/ui.
- Создать базовый Layout приложения (Mobile-first) с нижней панелью навигации (Bottom Navigation), содержащей 4 вкладки: "Покупки", "Дела", "Проекты", "Идеи".

### Этап 2: Интеграция Supabase и Авторизация
- Подключить Supabase клиент.
- Сгенерировать SQL-скрипт для создания таблиц в Supabase (profiles, purchases, tasks, projects, project_items, ideas) с включенным RLS (Row Level Security).
- Сделать простую страницу входа (Email/Password).

### Этап 3: Модуль «Покупки» (Быстрые списки)
- Реализовать UI компонента покупок.
- Настроить CRUD операции через Supabase.
- **Главная фича:** Реализовать логику зачеркивания и перемещения купленного товара в конец списка.
- Подключить Supabase Realtime для этого модуля.

### Этап 4: Модуль «Дела» (Рутина)
- Реализовать UI для To-Do листа.
- Разделить на "Активные" и "Выполненные".
- Настроить CRUD и Realtime подписку.

### Этап 5: Модуль «Идеи» (Бэклог)
- Простой CRUD для текстовых заметок.
- Отображение списком.

### Этап 6: Модуль «Проекты» (Сложные задачи)
- Реализовать страницу списка проектов.
- Реализовать внутреннюю страницу конкретного проекта.
- Добавить возможность привязывать вложенные элементы (задачи/покупки) к ID проекта.
- Вывод суммы ориентировочных затрат по проекту.

### Этап 7: Полировка и Багфикс
- Проверить работу всех Realtime подписок.
- Убедиться, что RLS в Supabase настроен безопасно (пользователи видят только данные своей "семьи"/группы).
- Финальная проверка мобильной верстки.

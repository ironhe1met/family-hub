# Техническое задание: Семейный веб-планировщик (MVP 1.0)

## 1. Описание продукта
Закрытое веб-приложение (Mobile-first) для мгновенной синхронизации семейных задач, списков покупок и планов. 
**Главная цель MVP:** Быстрый запуск, простота использования на ходу (со смартфона) и отображение изменений в реальном времени у обоих супругов.

## 2. Пользователи и Авторизация
* **Доступ:** Только по приглашению / заранее созданные аккаунты.
* **Роли:** Без разделения. Оба пользователя имеют полные права (Admin) на чтение, создание, редактирование и удаление (CRUD) любых записей.
* **Безопасность:** Базовая Email/Password авторизация. 

## 3. Архитектура модулей (Основной функционал)

Система разделена на 4 независимых логических блока:

### 3.1. Модуль «Покупки» (Быстрые чеклисты)
Списки для похода в магазин. Главный критерий — скорость работы одной рукой.
* **Группировка:** По названиям магазинов или категориям (например: «Сильпо», «Аптека», «Строительный»).
* **Структура записи:** * Название товара.
    * Количество / Объем.
    * Чекбокс (Куплено / Не куплено).
* **Логика:** При отметке «Куплено», товар зачеркивается и автоматически уходит в низ списка.

### 3.2. Модуль «Дела» (To-Do / Рутина)
Разовые задачи и бытовые поручения.
* **Структура записи:**
    * Краткое описание (например: «Оплатить коммуналку», «Забрать посылку»).
    * Чекбокс выполнения.
* **Логика:** Сортировка на «Активные» и «Выполненные». Возможность быстро смахнуть (swipe) задачу для удаления.

### 3.3. Модуль «Проекты» (Комплексные задачи)
Крупные цели, состоящие из множества шагов (например: «Обустройство детской площадки», «Ремонт»).
* **Структура проекта:**
    * Название проекта.
    * Текстовое описание / Заметки.
* **Вложенные элементы проекта:**
    * Суб-задачи или конкретные покупки, привязанные только к этому проекту.
    * Ориентировочная стоимость (UAH/USD).
    * Поле для URL (ссылки на товары, референсы).
    * Статус элемента (В процессе / Готово).

### 3.4. Модуль «Идеи / Бэклог»
Общий котел для планов на будущее, которые пока не требуют действий.
* Простой текстовый список (название + описание/ссылка), который можно в один клик конвертировать в «Дело» или «Проект».

## 4. Требования к UI/UX
* **Адаптивность:** Строго Mobile-first. Крупные чекбоксы, читаемые шрифты.
* **Навигация:** Нижнее меню (Bottom Navigation Bar) с переключением: `[ Покупки | Дела | Проекты | Идеи ]`.
* **Синхронизация:** Реалтайм-обновление. Изменения на одном устройстве должны мгновенно отображаться на другом без перезагрузки страницы.

## 5. Рекомендуемый стек технологий (Под генерацию ИИ)
Этот стек выбран для максимальной скорости разработки с помощью AI-ассистентов (Claude Code) и закрывает потребность в реалтайме из коробки.

* **Frontend:** Next.js (с App Router) + React. Это стандарт индустрии, Claude пишет на нем идеально.
* **Язык:** TypeScript. Строгая типизация поможет ИИ не путаться в структуре данных.
* **Стилизация и UI:** Tailwind CSS + shadcn/ui. Позволяет собрать красивый и адаптивный интерфейс готовыми блоками.
* **Backend, База данных и Реалтайм:** Supabase. 
    * Предоставляет готовую PostgreSQL базу данных.
    * Дает готовую авторизацию.
    * Поддерживает подписки на изменения в БД (Realtime), что идеально закроет требование синхронизации экранов.
* **Хостинг:** Vercel (для фронтенда) + Supabase (для бэка). 

## 6. Бэклог (План для Версии 2.0)
* Интеграция с Telegram (боты, уведомления о новых списках). *Техническая заметка: вебхуки из Supabase потом можно будет легко направить прямо в n8n для автоматизации.*
* Календарь, дедлайны и напоминания для модуля «Дела».
* Аналитика и подсчет общей суммы чека в «Покупках» и «Проектах».

# Discovery Brief

## 1. Суть проекта
- **Название:** Family Hub
- **Одно предложение:** Закрытое семейное веб-приложение (Mobile-first PWA) для совместного управления задачами, покупками, бюджетом, рецептами, идеями и проектами — с реалтайм-синхронизацией и делегированием между членами семьи.
- **Проблема:** Семейная организация разбросана по мессенджерам, заметкам и головам. Нет единого места для планирования, где оба супруга видят актуальную картину в реальном времени.
- **Решение:** Единый хаб с модульной архитектурой — каждый модуль закрывает свою область семейной жизни. Всё синхронизируется мгновенно, работает на телефоне одной рукой.

## 2. Целевая аудитория
- **Основная:** Семейная пара (двое взрослых), совместное ведение быта
- **Вторичная (будущее):** Дети (когда подрастут), расширенная семья
- **Потенциал:** Монетизация как SaaS для других семей (если продукт успешен)
- **Персоны:**
  - **Муж (владелец):** Технически грамотный, хочет автоматизацию, голосовой ввод, минимум ручной работы. Часто в движении — нужен быстрый доступ с телефона.
  - **Жена:** Активный пользователь для покупок и задач. Ценит простоту и визуальную понятность. Использует со смартфона в магазине, на кухне.

## 3. Ключевые функции (MVP)

### Модуль "Задачи" (приоритет 1)
- [ ] Три вида отображения: Kanban-доска, список, календарь
- [ ] Создание задач: название, описание, дедлайн (дата + время), приоритет (высокий/средний/низкий)
- [ ] Списки задач (группировка: "Дом", "Работа", "Дети" и т.д.)
- [ ] Делегирование: назначение задачи на члена семьи (кто создал, кто выполняет)
- [ ] Видимость: задачи "мои", "партнёра", "общие" — в одном интерфейсе
- [ ] Подзадачи — чеклист внутри задачи
- [ ] Повторяющиеся задачи (ежедневно / еженедельно / ежемесячно / кастом)
- [ ] Комментарии к задаче (обсуждение между членами семьи)
- [ ] Напоминания (за час, за день до дедлайна)
- [ ] Теги/метки для фильтрации
- [ ] Спринты — настраиваемые периоды (2 недели по умолчанию), привязка задач к спринту, просмотр прогресса за спринт
- [ ] Статусы: Новая → В работе → Выполнена → Архив
- [ ] Drag & Drop (мобильный + десктоп)
- [ ] Realtime-синхронизация

### Модуль "Покупки" (приоритет 2)
- [ ] Множественные списки (по магазинам: "Сільпо", "Аптека", "Строительный")
- [ ] Добавление товара: название, количество
- [ ] Чекбокс "Куплено" — зачёркивание, уход в низ списка
- [ ] Undo-удаление (4 секунды)
- [ ] Realtime-синхронизация
- [ ] Офлайн-режим: чекать товары без интернета, синк при восстановлении связи
- [ ] Простой и быстрый интерфейс — одна рука, в магазине на ходу

### Модуль "Идеи" (приоритет 3)
- [ ] Быстрые заметки: заголовок + описание
- [ ] Теги/категории
- [ ] Конвертация идеи → задача или → проект (в один клик)
- [ ] Поиск и фильтрация

### Модуль "Рецепты" (приоритет 4)
- [ ] CRUD: название, описание, ингредиенты, инструкции
- [ ] Фото рецепта
- [ ] Из рецепта → добавить ингредиенты в список покупок
- [ ] Поиск по рецептам

### Модуль "Проекты" (приоритет 5)
- [ ] Комплексные цели: название, описание, заметки
- [ ] Вложенные подзадачи со статусами
- [ ] Бюджет проекта: ориентировочная стоимость (UAH/USD), ссылки на товары
- [ ] Прогресс-бар по подзадачам
- [ ] Статусы: В работе / На паузе / Завершён

### Модуль "Бюджет" (приоритет 6)
- [ ] Доходы и расходы: ручной ввод, категории (еда, транспорт, жильё, подписки)
- [ ] Регулярные платежи: коммуналка, подписки, страховки (раз в месяц / квартал / год) с автонапоминаниями
- [ ] Запланированные расходы: что планируем купить/оплатить
- [ ] Баланс: сколько пришло, сколько ушло, сколько осталось
- [ ] Валюта: UAH (основная), USD (дополнительная)

### Общее для всех модулей
- [ ] Авторизация: email/password, JWT
- [ ] Профили пользователей с family_id (изоляция данных по семье)
- [ ] Dark mode + Light mode
- [ ] Тема: Google-стиль (Material Design 3)
- [ ] Mobile-first, touch-friendly
- [ ] i18n: украинский (основной) + английский
- [ ] PWA: установка на домашний экран
- [ ] Realtime-синхронизация между устройствами
- [ ] Навигация: логическая группировка модулей (а не россыпь вкладок)

### Навигация
Все 6 модулей видны сразу — одна строка, без вложенности. Связанные модули стоят рядом:

```
[ Задачи | Проекты | Покупки | Рецепты | Бюджет | Идеи ]
```

- Задачи + Проекты рядом (планирование)
- Покупки + Рецепты рядом (быт, кухня → магазин)
- Бюджет рядом с покупками (деньги)
- Идеи в конце (реже используется)

Один тап = модуль открыт. Конкретный UI (табы, иконки, расположение) — решает Design Agent.

## 4. Что НЕ входит в MVP
- Голосовой ввод (Telegram бот + n8n) → v2
- Telegram-уведомления → v2
- Push-уведомления (Service Worker) → v2
- Фото чеков → автоматическое распознавание и запись в расходы (модуль Бюджет) → v2
- Графики и аналитика бюджета → v2
- Модуль "Привычки" (ежедневный трекер) → v2
- Модуль "Заметки" (маркдаун, фото) → v2
- Модуль "Контакты" (семейная книга: врачи, мастера) → v2
- Модуль "Документы" (напоминания о сроках: паспорт, страховка, ТО) → v2
- Модуль "Здоровье" (лекарства, визиты, прививки) → v2
- Гостевой доступ по ссылке
- Экспорт PDF/CSV
- Аналитика и статистика
- Роли и разграничение прав (все = admin)
- Мультиязычность сверх UA/EN

## 5. Платформа и ограничения
- **Платформа:** Web (Mobile-first PWA)
- **Сервер:** Ubuntu, 4-8 GB RAM, 2-4 CPU, собственный VPS
- **Домен:** home.ironhelmet.com.ua (уже привязан)
- **База данных:** PostgreSQL (на своём сервере, БЕЗ Supabase)
- **Auth:** Свой (JWT + bcrypt), без сторонних сервисов
- **Realtime:** WebSocket (своя реализация)
- **Ограничения по команде:** Один разработчик + AI (Claude Code)
- **Ограничения по бюджету:** Минимальные расходы, всё на своём сервере

## 6. Аналоги и отличия
| Аналог | Что берём | Чем отличаемся |
|--------|-----------|----------------|
| Todoist | Kanban, приоритеты, повторяющиеся задачи, спринты | Семейный фокус, делегирование, общие списки |
| Google Tasks | Простота, интеграция с календарём | Больше модулей, realtime между членами семьи |
| AnyList | Списки покупок, шаринг | + задачи, проекты, идеи, рецепты — единый хаб |
| Notion | Модульность, гибкость | Проще, mobile-first, заточен под семью |
| Trello | Kanban-доска | + покупки, рецепты, бюджет — не только задачи |

## 7. Метрики успеха
- Оба супруга пользуются ежедневно (а не возвращаются в мессенджер)
- Нет багов, которые ломают соседние модули (проблема v1)
- Покупки можно чекать офлайн в подвале магазина
- Добавление задачи/покупки < 3 секунды
- Приложение готово к расширению новыми модулями без рефакторинга

## 8. Языки и локализация
- **Основной язык:** Украинский (uk)
- **Мультиязычность:** Украинский + Английский в MVP
- **Реализация:** Все строки через i18n, никакого хардкода
- **Расширение:** Другие языки — позже, при монетизации

## 9. Монетизация
- **MVP:** Не применимо — личное использование
- **Будущее:** Если продукт удачный — возможен SaaS (подписка для семей) или open-source с premium-фичами
- **Решение откладывается** до стабилизации и проверки на себе

## 10. Риски и допущения
- **Ключевые допущения:**
  - Два активных пользователя (муж + жена) на старте
  - Свой сервер справится с нагрузкой (2 пользователя, десятки записей)
  - WebSocket realtime можно реализовать без Supabase
  - PWA offline хватит для чекания покупок без сети
- **Риски:**
  - Без Supabase нужно самим делать auth, realtime, миграции — больше работы
  - Повторение проблемы v1: фикс одного модуля ломает другой → решение: тесты + модульная архитектура
  - Scope creep: много модулей → MVP чётко ограничен шестью
- **Регуляторные требования:** Не применимо (личное использование). При монетизации — GDPR

## 11. Открытые вопросы
- Нужен ли отдельный бэкенд (Node.js/Express/Fastify) или Next.js API Routes хватит?
- Какой WebSocket-фреймворк выбрать (Socket.io / ws / другой)? — решит Architect
- PWA offline: полный offline (все данные) или только покупки? — решит Architect
- Дизайн: точная цветовая палитра — решит Design Agent (Google-стиль подтверждён, конкретный акцентный цвет на усмотрение)

## Оценка сложности: M (Medium)

**Обоснование:**
- 6 модулей с CRUD + realtime
- Своя авторизация + WebSocket (вместо Supabase)
- PWA с офлайном
- Делегирование задач, спринты, повторяющиеся задачи
- Но: 2 пользователя, один сервер, один разработчик + AI

Между M и L. Для MVP с 6 модулями — **M**, при добавлении модулей v2 — станет **L**.

**Режим:** Полный (все 9 агентов)

## Handoff → Product Agent
- **Главный фокус:** Задачи — самый сложный и приоритетный модуль (3 вида, делегирование, спринты, повторы, подзадачи). Покупки — второй по важности (простые списки, офлайн). Бюджет — третий (регулярные платежи, запланированные расходы).
- **Сложные места:** Делегирование задач и видимость "мои/партнёра/общие". Спринты в семейном контексте (не перегрузить). Офлайн-синк для покупок. Realtime без Supabase. Логическая группировка навигации (6 модулей не должны превращаться в кашу).
- **Ограничения:** Свой сервер (Ubuntu, 4-8GB RAM). БЕЗ Supabase — свой PostgreSQL, auth, WebSocket. Один разработчик + AI.
- **v2 ключевое:** Фото чеков → OCR → автозапись в расходы. Telegram-бот + n8n для голосового ввода.
- **Оценка сложности:** M (Medium)
